# AI IELTS Speaking — 命令体系使用指南

不知道该用哪个？先调总管：`/master 你的需求`

---

## 一、Skills（参考手册型）

> 调用方式：`/skills` → 选择，或直接输入 `/master`、`/frontend` 等
> 特点：内容注入当前对话，写代码时随时参考，用完即走

### 总管
- **命令**: `/master`
- **功能**: 帮你分析需求、拆解任务、告诉你该调哪个 skill 或 agent。
- **场景**: 需求比较复杂，不知道从哪下手时。

### 前端
- **命令**: `/frontend`
- **功能**: 写页面、改组件、调样式，同时加载设计规范保证品质。
- **场景**: 任何跟页面、UI、交互、视觉相关的开发。

### 评分
- **命令**: `/scoring`
- **功能**: 优化 AI 评分 Prompt，让打分更稳定、反馈更有用。
- **场景**: 发现评分偏高/偏低、反馈空泛、Bad Case 需要分析时。

### 数据库
- **命令**: `/database`
- **功能**: 设计表结构、写迁移脚本、优化查询，管好 Supabase。
- **场景**: 需要加表、加字段、改 Schema、优化查询时。

### 治理
- **命令**: `/governance`
- **功能**: 开发后台的复核、申诉、Bad Case、数据看板等治理功能。
- **场景**: 做后台治理端的任何功能。

## 二、Subagents（独立执行型）

> 调用方式：`/agents` → 选择对应 agent
> 特点：独立窗口运行，不占主对话 token，跑完汇报结果

### 测试（test）
- **功能**: 跑构建、查类型错误、验证 API 和数据链路有没有坏。
- **场景**: 改完代码后，跑一遍确认没搞坏东西。

### 审查（review）
- **功能**: 检查代码质量、安全漏洞、性能问题、规范一致性。
- **场景**: 一个功能做完了，或者准备部署前，让它审一遍。

### 部署（deploy）
- **功能**: 搞定 Vercel 部署、环境变量配置、上线后排错。
- **场景**: 第一次部署、更新部署、或者线上出问题时。

### 日志（daily-log）
- **功能**: 一天结束时生成高密度的项目日志，记录改动、问题、知识点。
- **场景**: 每天收工时使用。

## 三、典型工作流

```
1. /master 我想加完整模考功能
   → 总管拆解任务，告诉你先调 database，再调 frontend...

2. /database 新增模考会话表
   → 数据库 skill 注入 Schema 参考，写 SQL、更新类型

3. /frontend 做模考页面
   → 前端 skill 注入设计规范，写页面、加动效

4. /scoring 调整模考评分权重
   → 评分 skill 注入评分标准，优化 Prompt

5. /agents → test
   → 测试 subagent 独立跑 build + 类型检查，回来报结果

6. /agents → review
   → 审查 subagent 独立检查安全和规范，回来出报告

7. /agents → deploy
   → 部署 subagent 搞定上线

8. /agents → daily-log
   → 日志 subagent 总结今天的所有改动
```

## 四、设计原则

| | Skill | Subagent |
|--|-------|----------|
| **运行方式** | 注入当前对话 | 独立窗口 |
| **适合场景** | 写代码时需要参考手册 | 跑完任务汇报结果 |
| **Token 消耗** | 占主对话 token | 不占主对话 token |
| **数量** | 5 个 | 4 个 |
